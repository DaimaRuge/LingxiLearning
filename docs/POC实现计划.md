## 一、POC 目标

### 1.1 核心验证点

| 验证点 | 成功标准 | 优先级 |
|--------|----------|--------|
| 文档导入与解析 | PDF/EPUB/MD 成功解析 | P0 |
| 向量索引与检索 | 100ms 内返回结果 | P0 |
| AI 对话（RAG）| 回答准确、引用溯源 | P0 |
| 知识图谱可视化 | 支持 100 节点渲染 | P1 |
| 多文档关联 | Connected Papers 风格 | P2 |

### 1.2 POC 范围

```
范围：MVP 核心功能的技术验证
- 文档处理流水线
- 向量检索系统
- RAG 对话系统
- 基础图谱展示

不包括：
- 用户系统
- 支付系统
- 多端同步
- 企业功能
```

## 二、技术方案

### 2.1 简化架构

```
┌─────────────────┐
│   Streamlit     │  Web UI (快速原型)
│   前端           │
└────────┬────────┘
         │ REST API
┌────────┴────────┐
│   FastAPI       │  后端服务
│   Backend       │
└────────┬────────┘
         │
    ┌────┴────┐
    │         │
┌───┴───┐ ┌───┴────┐
│ Qdrant │ │ GLM API│  向量存储  LLM
│(本地) │ │        │
└───────┘ └────────┘
```

### 2.2 技术选型（POC 阶段）

| 组件 | 选型 | 说明 |
|------|------|------|
| Web 框架 | Streamlit | 快速原型 |
| 后端 | FastAPI | Python 原生 |
| 向量数据库 | Qdrant (本地) | 轻量级 |
| LLM | GLM-5 API | 中文优化 |
| Embedding | BGE-large-zh | 中文向量化 |
| 图谱 | Pyvis / D3.js | 可视化 |

## 七、风险与应对

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| GLM API 调用失败 | 中 | 高 | 备用 GPT-4 API |
| 向量检索不准确 | 中 | 中 | 调优 Embedding 模型 |
| PDF 解析乱码 | 低 | 中 | 使用 OCR 兜底 |
| 图谱渲染卡顿 | 低 | 低 | 降级节点数量 |
