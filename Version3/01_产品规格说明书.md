## 一、产品定位

### 1.1 一句话定位

**灵犀智读** — 首款基于大模型 Agent 的个人终身学习数字伴侣

### 1.2 完整定位阐述

```
灵犀智读是一款面向终身学习者的 AI 原生知识管理产品。

它不是另一个笔记软件，而是"个人知识图书馆 + AI 学习伴侣 + 创作赋能平台"。

核心差异：
- 像图书馆一样组织知识（知识图谱）
- 像导师一样引导学习（AI 伴侣）
- 像思想家一样关联知识（Connected Papers）
- 像伙伴一样陪伴成长（进化记忆）
```

### 1.3 目标用户

| 用户类型 | 占比 | 核心需求 | 付费意愿 |
|----------|------|----------|----------|
| 学术研究者 | 30% | 文献管理、AI 辅助研究 | ¥200-500/月 |
| 技术开发者 | 30% | 代码知识库、AI 编程辅助 | ¥100-300/月 |
| 终身学习者 | 40% | 知识体系化、学习路径 | ¥50-150/月 |

### 2.2 核心模块详细规格

#### M1：分布式文档索引与关系重构

| 功能 | 规格 | 技术指标 |
|------|------|----------|
| **文档导入** | 支持 PDF/EPUB/MD/TXT/DOCX | - |
| PDF 解析 | PyMuPDF + OCR | 100 页/秒 |
| 扫描件 OCR | PaddleOCR 中文识别 | 准确率 95%+ |
| 文档命名 | AI 自动标准化命名 | 符合图书馆标准 |
| 全文索引 | 毫秒级搜索 | < 100ms 响应 |
| **文献关系图** | Connected Papers 风格 | 支持 10000+ 节点 |
| 参考文献抓取 | arXiv/Google Scholar/CNKI | 自动 DOI 解析 |

#### M2：知识网络扩展与完善

| 功能 | 规格 | 技术指标 |
|------|------|----------|
| 知识图谱 | Neo4j 图数据库 | 百万节点 |
| 节点类型 | 概念/人物/术语/文献/代码 | 自动实体识别 |
| 关系抽取 | NLP 自动关系抽取 | F1 > 85% |
| 图谱可视化 | D3.js 交互图谱 | 60fps 渲染 |
| 智能推荐 | 基于图谱的协同过滤 | CTR > 20% |

#### M3：学习与创作辅助

| 功能 | 规格 | 技术指标 |
|------|------|----------|
| AI 对话 | 基于 RAG 的问答 | 引用准确率 90%+ |

#### M4：个人终身学习伴侣

### 5.2 核心 API 设计

#### 文档管理 API

```yaml
POST   /api/v1/documents          # 导入文档
GET    /api/v1/documents           # 获取文档列表
GET    /api/v1/documents/{id}     # 获取文档详情
DELETE /api/v1/documents/{id}     # 删除文档
PUT    /api/v1/documents/{id}     # 更新文档元数据
GET    /api/v1/documents/{id}/content  # 获取文档内容
POST   /api/v1/documents/{id}/chat     # 对文档提问
```

#### 知识图谱 API

```yaml
GET    /api/v1/graph               # 获取全局图谱
GET    /api/v1/graph/nodes/{id}   # 获取节点详情
GET    /api/v1/graph/related/{id} # 获取相关节点
POST   /api/v1/graph/search       # 图谱搜索
```

#### AI 伴侣 API

```yaml
POST   /api/v1/companion/chat    # AI 对话
GET    /api/v1/companion/memory   # 获取记忆
POST   /api/v1/companion/learn    # 主动学习触发
GET    /api/v1/companion/profile  # 获取成长档案
```
