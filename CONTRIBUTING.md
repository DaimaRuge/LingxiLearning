# 灵犀智读 - 开发贡献指南

## 一、项目结构

```
LingxiLearning/
├── .github/
│   └── workflows/          # GitHub Actions
├── docs/                  # 项目文档
├── Version1/              # V1.0 分析文档
├── Version2/             # V2.0 原型文档
├── Version3/              # V3.0 规格文档
└── README.md
```

## 二、文档贡献

### 文档命名规范

```
{序号}_{文档名称}.md
示例：01_需求深度分析.md
```

### Markdown 规范

- 标题使用 ## 
- 表格使用标准 Markdown 表格
- 代码块标注语言
- 中文标点符号

## 三、版本管理

### 版本标签

```
V1.x - 需求分析阶段
V2.x - 产品设计阶段  
V3.x - 技术规格阶段
```

### Commit 规范

```
{类型}: {简短描述}

类型：
- feat: 新功能
- docs: 文档更新
- fix: 问题修复
- refactor: 重构
- test: 测试
```

## 四、问题反馈

请通过 GitHub Issues 提交问题，包含：

1. 问题描述
2. 复现步骤
3. 预期行为
4. 实际行为
5. 环境信息
